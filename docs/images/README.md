# 📸 图片存放说明

这个目录用于存放 README 文档中的截图和图片。

## 📋 需要的截图列表

### 1. **logo.png**
- **尺寸建议**: 200x200 或更大（正方形）
- **内容**: 项目Logo（如果有的话）
- **用途**: README顶部展示

### 2. **main-dashboard.png**
- **尺寸建议**: 1200x800
- **内容**: 主界面/仪表盘截图
- **用途**: 展示主要界面

### 3. **init-wizard.png**
- **尺寸建议**: 1200x800
- **内容**: 项目初始化向导界面
- **用途**: 展示一键初始化功能

### 4. **project-list.png**
- **尺寸建议**: 1200x800
- **内容**: 项目列表界面
- **用途**: 展示项目管理功能

### 5. **settings.png**
- **尺寸建议**: 1200x800
- **内容**: 设置界面（包括GitHub Token配置等）
- **用途**: 展示配置功能

### 6. **logs.png**
- **尺寸建议**: 1200x800
- **内容**: 实时日志界面
- **用途**: 展示WebSocket日志推送功能

### 7. **sync-demo.gif** (可选)
- **尺寸建议**: 800x600
- **文件大小**: < 5MB
- **内容**: 自动同步的动画演示
- **用途**: 展示核心自动备份功能

---

## 💡 截图技巧

1. **使用暗黑主题**：项目是太空风格，暗黑主题更好看
2. **窗口大小**：调整合适的窗口大小，不要太小
3. **数据准备**：创建一些示例项目和日志，让界面更丰富
4. **高清截图**：建议使用 2x 显示倍率截图（视网膜屏幕效果）
5. **去除敏感信息**：如果有GitHub Token或个人信息，记得打码

---

## 📐 图片命名规范

- 使用小写字母和连字符：`main-dashboard.png`
- 不要使用空格或中文：~~`主界面 截图.png`~~
- 使用有意义的名称：`init-wizard.png` ✅  而不是 `screenshot1.png` ❌

截图完成后，将图片直接放到这个目录下即可！
